# Benchmark Datasets

This document describes the datasets used for kmerdet benchmarking, how to
prepare them, and the ground truth file format.

---

## Dataset Types

### 1. Simulated Datasets (Recommended for Development)

Generated by `generate_simulated.py`. Require no real patient data, no
jellyfish installation, and run within seconds. Suitable for CI regression
testing.

| Dataset | Contents | Purpose |
|---------|----------|---------|
| `simulated_snv_indel` | 50 SNVs + 30 INDELs at VAF 0.05%–50% | Core accuracy regression |
| `simulated_large_indels` | 20 large INDELs (5–50 bp) | INDEL sensitivity testing |
| `simulated_ultra_low_vaf` | 40 variants at VAF 0.01%–0.1% | LOD characterization |

Quick generation:
```bash
python3 docs/benchmarking/datasets/generate_simulated.py \
    --output-dir /tmp/kmerdet-simulated \
    --preset all
```

### 2. Dilution Series (LOD Characterization)

A dilution series mixes tumor k-mer counts with normal k-mer counts at known
fractions. This directly characterizes the limit of detection (LOD) as a
function of VAF.

Preparation requires:
- A high-purity tumor jellyfish database (ideally from tumor DNA at 100% purity)
- A matched normal jellyfish database (cfDNA from the same patient timepoint,
  before tumor recurrence, or a healthy donor)
- Target FASTA files from the tumor-informed panel

### 3. Real Clinical Data

Patient samples with known variants from a primary tumor biopsy. Ground truth
is established by comparison with:
- Tumor WGS or targeted NGS (high VAF)
- Orthogonal assay (ddPCR or concordant alignment-based pipeline)

See the `patient_cohort_10` dataset entry in `framework/benchmark_config.toml`
for the expected directory structure.

## Ground Truth File Format

The ground truth TSV file specifies every variant that should be detected (or
should not be detected, for specificity testing). One variant per row.

### Column specification

| Column | Type | Description |
|--------|------|-------------|
| `chrom` | string | Chromosome name (e.g., `chr1` or `1`). Case-insensitive; `chr` prefix optional. |
| `pos` | int | 1-based genomic position of the variant (leftmost base of the change). |
| `ref` | string | Reference allele in uppercase DNA bases. Use `-` for pure insertions. |
| `alt` | string | Alternative (variant) allele in uppercase DNA bases. Use `-` for pure deletions. |
| `type` | string | Variant type. Must match kmerdet output type: `Substitution`, `Insertion`, `Deletion`, `ITD`, `Complex`. |
| `true_vaf` | float | True variant allele frequency as a fraction (e.g., `0.01` for 1%). Set to `0.0` for absent variants (specificity controls). |
| `category` | string | Optional grouping label (e.g., `SNV`, `tier1`, `homopolymer`). Leave blank if not needed. |

### Header row

```
chrom	pos	ref	alt	type	true_vaf	category
```

### Example

```tsv
chrom	pos	ref	alt	type	true_vaf	category
chr7	55174772	T	G	Substitution	0.05	SNV_tier1
chr7	55174772	T	G	Substitution	0.001	SNV_low_vaf
chr17	7577538	C	T	Substitution	0.00	absent_control
chr11	534287	G	GACGT	Insertion	0.02	small_indel
chr2	209113192	ACAGTAGCATCAT	A	Deletion	0.10	large_deletion
chrX	100620877	ATTTG	ATTTGATTTG	ITD	0.15	tandem_dup
```

### Notes on variant representation

1. **SNVs**: `ref` and `alt` are single bases. `pos` is the 1-based position of
   the changed base. Example: `chr1  100  A  T  Substitution  0.05`.

2. **Insertions**: `ref` is the base before the insertion (anchor base), `alt`
   is the anchor base followed by the inserted sequence. Example:
   `chr1  100  G  GACC  Insertion  0.01` represents inserting `ACC` after position 100.

3. **Deletions**: `ref` is the anchor base followed by the deleted sequence, `alt`
   is just the anchor base. Example: `chr1  100  GACGT  G  Deletion  0.02`.

4. **ITDs (In-tandem duplications)**: Encoded like insertions where the inserted
   sequence is a repeat of adjacent reference sequence.

5. **Chromosome normalization**: kmerdet treats `chr1` and `1` as identical.
   Use whichever form matches your reference genome.

6. **Absent variants** (`true_vaf = 0.0`): Include these as specificity controls
   when you have a priori knowledge that a variant is not present (e.g., variants
   from a different patient's panel run against this sample).

## Preparing Input Data: Real Samples

### Step 1: UMI deduplication

```bash
humid --umi-len 12 -o deduplicated.fastq.gz sample_R1.fastq.gz sample_R2.fastq.gz
```

HUMID produces a single deduplicated FASTQ. The `--umi-len` should match your
library preparation protocol (typically 12 or 16 bp).

### Step 2: Jellyfish k-mer counting

```bash
jellyfish count \
    -m 31 \
    -C \
    -s 500M \
    -t 8 \
    -L 2 \
    -o sample.jf \
    deduplicated.fastq.gz
```

Parameters:
- `-m 31`: k-mer length (must match kmerdet's `--kmer-length` default of 31)
- `-C`: canonical mode (count forward and reverse complement together)
- `-s 500M`: initial hash size; increase for large libraries
- `-t 8`: threads
- `-L 2`: minimum count (discard k-mers seen only once)

### Step 3: Target FASTA files

Target FASTA files are created by extracting the genomic region around each
known variant, typically ±35 bp (the thesis used `multiseqex --flank 35`).

Format: one FASTA file per target, or all targets in one multi-FASTA file.
The target sequence must include enough unique flanking sequence to serve as
anchor k-mers for the walking algorithm.

```
>EGFR_L858R
GCAGCTCATGCCCTTCGGCTGCCTCCTGGACTATGTCCGGGAACACAAAGACAATATTGGCTCCCAGTAC
```

### Step 4: Ground truth

Create the ground truth TSV manually from clinical/molecular findings, or
generate it automatically from VCF files:

```bash
# Convert a VCF with AF field to ground truth TSV
python3 - <<'EOF'
import sys
print("chrom\tpos\tref\talt\ttype\ttrue_vaf\tcategory")
for line in sys.stdin:
    if line.startswith("#"):
        continue
    fields = line.strip().split("\t")
    chrom, pos, _, ref, alt, _, _, info = fields[:8]
    # Extract AF from INFO field (adjust field name to match your VCF)
    af = next((f.split("=")[1] for f in info.split(";") if f.startswith("AF=")), "0")
    vtype = "Substitution" if len(ref) == len(alt) == 1 else (
        "Insertion" if len(alt) > len(ref) else "Deletion"
    )
    print(f"{chrom}\t{pos}\t{ref}\t{alt}\t{vtype}\t{af}\t")
EOF < variants.vcf > ground_truth.tsv
```

## Public Datasets

The following public datasets could be used for benchmarking, though each
requires data access agreements and preprocessing:

### SEQC2 Somatic Mutation Truth Set
- **URL**: https://ftp.ncbi.nlm.nih.gov/ReferenceSamples/seqc/Somatic_Mutation_WG/
- **Variants**: Validated SNVs and INDELs in Genome in a Bottle (GIAB) cell lines
- **Use case**: Baseline accuracy benchmarking
- **Note**: Not liquid biopsy; requires adjusting expected VAF (typically ~50%)

### COLO829 ctDNA Reference Material
- **URL**: Various commercial providers (Horizon Discovery, SeraCare)
- **Variants**: Known somatic variants in COLO829 melanoma cell line, diluted into
  normal cfDNA at defined VAF fractions (5%, 1%, 0.1%)
- **Use case**: LOD characterization, direct comparison with published ctDNA methods

### Pan-Cancer TCGA cfDNA Simulation
- **Reference**: Adalsteinsson et al., Nature Communications 2017
- **Approach**: In silico fragmentation of tumor WGS reads to simulate cfDNA at
  known VAF fractions
- **URL**: dbGaP accession phs001273

### Illumina NIPT Reference Sample
- **Vendor**: Illumina
- **VAF**: Low (0.1–1% fetal fraction)
- **Note**: Different context (NIPT vs. cancer), but useful for testing ultra-low-VAF behavior

## Simulated Data Generation

The `generate_simulated.py` script creates synthetic benchmark data without
requiring real jellyfish databases. See the script's `--help` output for all
options.

### How simulation works

1. **Construct synthetic reference sequences**: Each target is a random DNA
   sequence of configurable length (default: 150 bp) with low homopolymer
   frequency to ensure uniqueness.

2. **Inject variants**: For each variant, mutate the reference sequence at a
   random position to create the alternate allele (SNV, insertion, deletion, or ITD).

3. **Simulate k-mer counts**: Compute all k-mers from the reference and alternate
   sequences. Assign counts according to a Poisson distribution with mean
   `coverage * vaf` for variant k-mers and `coverage * (1 - vaf)` for reference
   k-mers. Add Poisson noise to model sequencing.

4. **Write mock database**: Save k-mer → count mappings to a TSV file that
   kmerdet can read in place of a real jellyfish database.

5. **Write ground truth**: Record the injected variants with their true VAFs.

### Simulation caveats

- Does not model GC bias, repetitive regions, or sequencing error hotspots.
- All k-mers are assumed to be unique (no off-target counts).
- Coverage is uniform across the target sequence.
- Results on simulated data will be more optimistic than real data.

Use simulated data for:
- Algorithm correctness testing (CI)
- Parameter sweep exploration
- Performance benchmarking (timing, memory)

Use real data for:
- Clinical validity assessment
- Comparison with published sensitivity numbers
